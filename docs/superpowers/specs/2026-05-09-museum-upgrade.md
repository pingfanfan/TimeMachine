# 《时光档案馆》设计升级文档(v2)

> 在原 v1 spec(`2026-04-23-shiguangda-design.md`)基础上的产品定位升级。
> v1 仍然有效,本文档覆盖 v1 中改动的章节;未提及的章节(API 接入、错误处理、风险等)沿用 v1。
> 升级日期:2026-05-09

## 一、产品重新定位

### 旧定位(v1)
**《时光答》** — 知乎编年观点演变史。一个"输入老话题 → 看时光机"的**工具**。

### 新定位(v2)
**《时光档案馆》**(Zhihu Chronicle · 知乎思想编年志)— AI 时代的知乎思想博物馆。

**变化的本质:** 从"工具"升级为"内容机构"。用户不是来用工具,而是来"逛档案馆"。每一个话题不是"搜索结果",而是一份"档案条目"。每一个专辑不是"功能模块",而是一次"策展"。

### 为什么这个定位升级有杠杆

| 维度 | 工具定位(v1) | 档案馆定位(v2) |
|---|---|---|
| 用户心智 | "我有问题,想看演变" | "知乎十五年的思想档案,我来逛逛" |
| 评分:创新度(30%) | "做了个时光机功能" | "AI 时代,把知乎做成了思想档案馆" |
| 评分:AI 场景价值(30%) | "AI 帮我看观点" | "AI 是策展人,把十五年讨论组织成有意义的档案" |
| 路演冲击力 | 演示 1 个话题 | 演示一座博物馆 |
| 传播性 | "这个工具不错" | "我刚去了知乎档案馆" |
| 商业化空间 | 单点工具 | 策展人生态 + 订阅会员 + 主题展览 |
| 完成度风险 | 低 | 中(但用 IA 简化抵消) |

## 二、新核心体验闭环

### 首访 30 秒心路(博物馆版)

1. **打开首页**(档案馆 Foyer)
   - 顶部:深色巨幕 +「时光档案馆 · 知乎思想编年志」白色衬线大字
   - 居中:小字 "馆藏 30 份档案 / 481 份金句 / 跨越 15 年"(实时数字)
   - 下面:**今日入档**(今日热榜话题升级版,带"入档章"动画)
2. **下滑到三大策展入口**(三张大卡)
   - 🅰 **【主题档案】** "《35 岁的中国人:十五年焦虑史》等 6 份主题策展"
   - 🅱 **【年代切片】** "2011-2026 每一年的中国人在想什么"
   - 🅒 **【金句博物馆】** "481 句穿越十五年的精华"
3. **下滑到精选档案条目**(原瀑布流改名)
   - 12 张档案卡(原话题卡升级):每张带"档案编号 ARCHIVE-035"、印章式分类标签、`2015 vs 2026` 对照金句
4. **底部:策展人手记**
   - 一段 200 字短文,模拟"本馆策展人本周观察",签名「策展人 · 看山」
5. **顶部 nav 始终带搜索框**(轻提示:搜索一份档案…)

### 进入主题专辑(以《35 岁的中国人》为例)

1. 全屏深色封面 +「档案 THEME-001」编号 + 标题大字 + 副标题 + 策展人导语
2. 下滑列出 4 个相关话题档案条目(可点入各自时间轴)
3. 底部"导出本专辑海报"(整组打包成竖版长图,1080×2880,4 个话题串读式呈现)

### 进入金句博物馆

1. 顶部筛选条:【按年份】【按情绪】【按主题】【随机】
2. 瀑布流式金句卡(每卡 = 一句金句 + 年份 + 话题 + 作者)
3. 点击金句 → 跳到所属话题的对应年份卡
4. 顶部"今日金句墙"模式(40 句无限滚动,适合截图分享)

## 四、数据模型扩展(在 v1 基础上)

```sql
-- 给所有原表加档案编号
ALTER TABLE topics ADD COLUMN archive_no TEXT;
-- e.g. "ARCHIVE-035"

-- 新增专辑表
CREATE TABLE collections (
  id TEXT PRIMARY KEY,           -- slug, e.g. "35-of-china"
  title TEXT NOT NULL,
  subtitle TEXT,
  archive_no TEXT NOT NULL,      -- "THEME-001" / "DECADE-2020" / "QUOTE-MUSEUM"
  category TEXT NOT NULL,        -- theme / decade / mood
  curator_intro TEXT,            -- 编辑导语 200-400 字
  curator_name TEXT DEFAULT '看山',  -- 策展人署名
  cover_image_url TEXT,
  display_rank INT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE collection_topics (
  collection_id TEXT REFERENCES collections(id) ON DELETE CASCADE,
  topic_id TEXT REFERENCES topics(id) ON DELETE CASCADE,
  topic_rank INT,
  curator_note TEXT,             -- 该话题在专辑里的策展批注
  PRIMARY KEY (collection_id, topic_id)
);

-- 策展人手记(首页底部)
CREATE TABLE curator_notes (
  id SERIAL PRIMARY KEY,
  title TEXT,
  body TEXT,                     -- 200-400 字短文
  publish_date DATE,
  curator_name TEXT DEFAULT '看山'
);
```

**金句博物馆**:不需要新表,直接对 `year_summaries.golden_quote` 全表查询 + 按年份/主题/情绪聚合(情绪可由 LLM 提前打标存到 year_summaries.mood 字段)。

```sql
ALTER TABLE year_summaries ADD COLUMN mood TEXT;
-- e.g. "焦虑" / "希望" / "戏谑" / "悲怆" / "理性"
```

## 五、视觉设计语言升级

### 色板(深化博物馆感)

| Token | 旧 (v1) | 新 (v2) | 用途 |
|---|---|---|---|
| `ink` | `#0B1026` | `#08101F` | 主背景:更深的"档案室深蓝" |
| `paper` | `#F4E4BA` | `#EBE0C4` | 文字/卡片:稍偏暖的"旧档案纸" |
| `stamp` | `#E63946` | `#A8252F` | 印章红:饱和度降低,"老印章"感 |
| `mute` | — | `#5A6478` | 辅助文字 |
| `highlight` | — | `#F1B644` | 高亮(入档章背景) |

### 字体

| 用途 | 字体 |
|---|---|
| 标题 / 大字 / 引言 | Noto Serif SC(衬线,博物馆牌匾感) |
| 正文 / 标签 | Noto Sans SC(无衬线) |
| 档案编号 / 元数据 | Iosevka 或 JetBrains Mono(等宽,档案编码感) |

### 视觉新元素

| 元素 | 描述 | 出现位置 |
|---|---|---|
| **入档章** | 红印章风,内嵌"入档"或档案编号 | 话题卡角、专辑封面、海报水印 |
| **档案编号** | `ARCHIVE-XXX` / `THEME-XXX` 等等宽字体 | 每个内容单元的元数据 |
| **策展人署名** | "策展人 · 看山" | 编辑导语下方 |
| **入档动画** | 卡片淡入 + 红印章盖章 fade-in(800ms) | 滚动到视口时触发 |
| **空间留白** | section padding 从 `py-16` 提升到 `py-24` | 全局 |

### 海报模板升级

旧:杂志感长图
新:**档案副本风**(Document Archive Style)
- 顶部:"档案 ARCHIVE-035 · 馆藏 第 035 号"
- 中部:话题标题 + 时间轴 6 个金句
- 底部:策展人署名 + 入档章 + `知乎时光档案馆 · #知乎黑客松2026`
