# 《时光答》设计文档

> 知乎 Hackathon 2026 · AI 脑洞实验室参赛作品
> 赛道:灵感引擎(主)+ 引力场(次,分享传播维度)
> 起稿日期:2026-04-23

## 一、产品定位

**《时光答》**:知乎编年观点演变史。

输入一个话题,从 2011 年到 2026 年的观点演变被时间轴式展开,你能看到"同一个问题,不同时代的中国人是怎么想的";滑到底部,AI 基于趋势外推给出 2029 年的三种未来情境;一键生成一张《观点十年编年史》长图海报,分享到知乎想法。

### 定位三要素

| 维度 | 定位 |
|---|---|
| 用户 | 吃瓜读者 + 爱分享的年轻用户(二次传播驱动) |
| 情绪锚点 | "哇,原来 2015 年大家都是这么想的,好有意思!" + "这张图我想发想法" |
| 商业/评分价值 | 独占知乎 15 年时间纵深这一护城河,别的平台无法复刻 |

### 为什么这个 idea 评分会高

- **AI 场景价值(30%)**:AI 真正被用来做"纵向时代分析",不是另一个写作助手。知乎 15 年数据是训练大模型都摸不到的原始材料。
- **创新度(30%)**:市面上所有 AI 产品都在做"回答一个问题",没人做"看一个问题是怎么被回答 15 年的"。时间维度是全新视角。
- **完成度(20%)**:架构清晰,批量预处理 + 前端只读缓存,48h 可以稳稳跑完。
- **设计感(15%)**:时间轴滚动 + 海报形态天生出片。
- **传播性**:海报 + 知乎想法发布钩子,自带二次曝光闭环,附送人气奖加成。

## 二、核心体验闭环

### 首访 30 秒心路

1. **打开首页**:首屏是一张大 banner —— "今日热榜话题 · 时代前世今生"(Cron 每 4 小时从知乎热榜自动选一个有时代纵深的话题推为今日头条),占整个视口高度 60%,顶部留导航和搜索框。
2. **下滑进瀑布流**:banner 下方是 30+ 精选话题的 3 列瀑布流,每张卡封面印着"2015 vs 2026"对照金句(例:「程序员 35 岁后会怎样」—— 2015 「转管理或转架构」 vs 2026「进厂送外卖或被 AI 替代」)。用户产生点击冲动。
3. **进入话题页**:纵向时间轴 2011 → 2026,每年一张卡。滚动视差动画 + 事件锚点气泡("2020 · 疫情爆发""2022 · ChatGPT 发布""2024 · 互联网大裁员")。读到某年,该年的 1-2 个主流观点 + 代表金句 + 代表答主。
4. **滚到底部**:**2029 预测章节** —— AI 给 3 种未来情境(保守 / 主流 / 激进)。
5. **点击右下浮动按钮**:3 秒生成一张竖版海报,时间轴金句精华版 + 2029 预测 + `#知乎黑客松2026` 水印 → 一键发布到知乎想法(通过圈子发帖 API)。

### 二次使用钩子

- 首页每日更新"今日热榜话题 · 时代前世今生" banner(Cron 触发)
- 订阅某话题,话题有新热度 / 有新回答时 push 邮件
- 海报的朋友圈 / 想法曝光为产品带来新用户点进首页

## 三、系统架构

### 三层分层

```
┌────────────────────────────────────────────┐
│  前端层 (Next.js 14 App Router)            │
│  - 首页 / 话题页 / 海报页 / 搜索兜底       │
│  - Framer Motion 滚动动画                  │
│  - 响应式 Web(移动端友好)                │
└────────────────────────────────────────────┘
                    ↕
┌────────────────────────────────────────────┐
│  API 层 (Next.js Route Handlers)           │
│  - /api/topics/list       (读缓存)          │
│  - /api/topics/[id]       (读缓存)          │
│  - /api/search            (代理 + 缓存)     │
│  - /api/predict           (代理直答 Agent)  │
│  - /api/poster            (satori 生成 PNG) │
│  - /api/publish           (代理圈子发帖)    │
└────────────────────────────────────────────┘
                    ↕
┌────────────────────────────────────────────┐
│  数据层                                      │
│  - Postgres/Supabase     结构化数据          │
│  - 本地 JSON 缓存        精选话题冷启动数据   │
│  - Redis (可选)          搜索/预测运行时缓存   │
└────────────────────────────────────────────┘
                    ↑
┌────────────────────────────────────────────┐
│  离线数据管线 (Python 脚本 · 赛前预处理)     │
│  1. 话题精选 (30 个手工 + 20 个 AI 推荐)    │
│  2. 抓取 (调用 /api/v1/content/zhihu_search) │
│  3. 按年分桶                                │
│  4. 观点聚类 (embedding + LLM)              │
│  5. 金句提炼 (LLM)                         │
│  6. 事件锚点标注 (LLM + 手工校对)           │
│  7. 预生成 2029 预测                        │
│  8. 写入 Postgres + 冷启动 JSON             │
└────────────────────────────────────────────┘
```

### 关键组件

#### C1:话题精选器(离线)

- **输入**:人工候选池(30 个有时代张力的问题)+ 从热榜 API 拉取的 Top 100(近一周)
- **输出**:锁定 30 个入围话题,存 `topics` 表
- **筛选维度**:① 至少有 15 年答案沉淀 ② 主流观点在不同年代有明显差异 ③ 有传播点(能激起共鸣/争议)
- **候选主题方向**(示例):职场焦虑 / 婚恋彩礼 / 生育 / 买房 / 副业 / 出国 / 消费降级 / 相亲 / 互联网行业 / AI 冲击 / 健身 / 读书 / 理财 / 35 岁危机 / 考公考研 / 润学 / 恋爱观 / 独居 / 抑郁 / 算命玄学 / 体制内 / 短视频 / 打工人 / 原生家庭…

#### C2:历史回答抓取器(离线)

- **输入**:话题关键词 + 话题锁定的 question_id(如有)
- **执行**:调用 `GET /api/v1/content/zhihu_search`,按时间窗口分页(2011-2026,每年单独查一次)
- **控速**:每个话题耗 API 10-20 次,30 话题 × 15 = 450 次,刚好在 1000/天 限额内,**一天跑完所有话题**
- **输出**:每个话题 200-500 条回答,存 `answers` 表(含 content / year / like_count / author / timestamp)

#### C3:观点聚类 pipeline(离线)

- **每年一桶**:同一年的所有回答,用 embedding 做聚类(K=3-5),抽出每一类的中心答案
- **LLM 提炼**:喂中心答案给 LLM,生成该年的:
  - 1-2 个主流观点(1 句话)
  - 1 个代表金句(从真实回答中抽取或改写,带出处)
  - 时代特征注释(用 1 句话点出这一年的社会背景)
- **模型**:主力 DeepSeek-V3(便宜),质量检查用 Claude Haiku 4.5
- **输出**:每年一条 `year_summary` 记录

#### C4:事件锚点标注

- 时间轴上固定的历史事件节点(预置 15-20 个):
  - 2013 微信崛起
  - 2015 双创热 / A 股股灾
  - 2016 知识付费元年
  - 2018 互联网寒冬 / 中兴禁令
  - 2019 996.ICU
  - 2020 新冠爆发
  - 2021 双减 / 共同富裕
  - 2022 俄乌 / 互联网大厂裁员 / ChatGPT 发布
  - 2023 大模型爆发
  - 2024 AI 替代讨论白热化 / 互联网二次大裁员
  - 2025 AI Agent 元年 / 具身智能量产 / 一线城市白领二次裁员
  - 2026 当下(Hackathon 当年,由 AI 实时从最新热榜归纳当年锚点)
- 每个话题展示时,AI 根据话题相关性**挑 4-6 个锚点**插入时间轴对应年份

#### C5:2029 预测生成器

- **输入**:一个话题的完整时间轴摘要
- **LLM Prompt**(节选):
  > 基于这个话题 2011-2026 年的观点演变轨迹,请外推 2029 年最可能的三种情境:
  > 1. 保守情境(当前趋势平稳延续)
  > 2. 主流情境(综合各信号的最可能未来)
  > 3. 激进情境(小概率但不可忽视的尾部风险)
  > 每种情境 30-50 字,带一句代表金句。明确标注"此为 AI 基于公开讨论外推的推演,请勿作决策依据"。
- **模型**:运行时调用**知乎直答 Agent**(100 次/天),自带知乎语料背书,回答更有说服力。失败降级到 Haiku。
- **缓存**:同一话题 7 天内不重跑,减少 API 消耗。

#### C6:海报生成器

- **工具**:Vercel satori(React → SVG → PNG)
- **模板**:竖版 1080×1920
  - 顶部:话题问题(大字)
  - 中部:6 个时间节点,每节点"年份 + 金句"
  - 下部:2029 预测(选主流情境)
  - 底部:品牌水印 + `#知乎黑客松2026` + 二维码扫码看完整时间轴
- **输出**:PNG,支持直接下载 + 调用知乎圈子发帖 API 一键发布

#### C7:搜索兜底通道

- 首页顶部搜索框
- 用户输入话题 → 先查精选库(编辑距离模糊匹配)→ 命中即跳话题页
- 未命中 → 实时走简化 pipeline(直答 API 1 次 + 站内搜索 API 3 次,生成"浅层时光轴",标注"精选版正在生成,先给你一个快速版")
- 搜索词进入"用户请求队列",赛后人工评审选下一批精选

## 四、数据模型

```sql
-- 话题
CREATE TABLE topics (
  id TEXT PRIMARY KEY,           -- slug, e.g. "programmer-35"
  title TEXT,                    -- 标题问题
  question_id TEXT,              -- 知乎 question_id (如已锁定)
  cover_quote_old TEXT,          -- 封面对照金句(旧)
  cover_quote_new TEXT,          -- 封面对照金句(新)
  category TEXT,                 -- 职场 / 婚恋 / 消费 / ...
  featured_rank INT,             -- 精选排序,NULL 表示搜索兜底生成
  created_at TIMESTAMPTZ
);

-- 年度观点摘要
CREATE TABLE year_summaries (
  topic_id TEXT REFERENCES topics(id),
  year INT,                      -- 2011..2026
  primary_view TEXT,             -- 主流观点 1
  secondary_view TEXT,           -- 主流观点 2(可空)
  golden_quote TEXT,             -- 代表金句
  quote_author TEXT,             -- 答主(可空)
  quote_source_url TEXT,         -- 原回答链接
  era_caption TEXT,              -- 时代特征注释
  PRIMARY KEY (topic_id, year)
);

-- 事件锚点
CREATE TABLE events (
  id SERIAL PRIMARY KEY,
  year INT,
  title TEXT,                    -- 如 "ChatGPT 发布"
  description TEXT,
  category TEXT                  -- 用于话题匹配
);

CREATE TABLE topic_events (
  topic_id TEXT REFERENCES topics(id),
  event_id INT REFERENCES events(id),
  PRIMARY KEY (topic_id, event_id)
);

-- 未来预测
CREATE TABLE predictions (
  topic_id TEXT REFERENCES topics(id),
  target_year INT,               -- 默认 2029
  scenario_conservative TEXT,
  scenario_mainstream TEXT,
  scenario_radical TEXT,
  generated_at TIMESTAMPTZ,
  PRIMARY KEY (topic_id, target_year)
);

-- 原始回答(仅保留抓取数据,供 pipeline 用,不对外暴露)
CREATE TABLE raw_answers (
  id BIGSERIAL PRIMARY KEY,
  topic_id TEXT,
  year INT,
  author TEXT,
  content TEXT,
  like_count INT,
  url TEXT,
  fetched_at TIMESTAMPTZ
);
```

## 五、关键 API 设计

| 路径 | 方法 | 功能 | 缓存 |
|---|---|---|---|
| `/api/topics/list` | GET | 首页瀑布流 + 今日热榜话题 | ISR 1h |
| `/api/topics/[id]` | GET | 单话题完整时间轴数据 | ISR 24h |
| `/api/search` | POST | 模糊匹配 + 兜底生成 | 按 query 字符串 LRU 7d |
| `/api/predict/[id]` | POST | 运行时刷新 2029 预测 | per-topic 7d |
| `/api/poster/[id]` | GET | 生成海报 PNG | CDN 边缘缓存 |
| `/api/publish` | POST | 调用知乎圈子发帖接口(需 OAuth) | 不缓存 |
| `/api/hotlist-today` | GET | 今日热榜话题(cron 刷新) | cron 每 4h |

## 六、知乎平台能力接入

| API | 使用点 | 额度消耗估算 |
|---|---|---|
| `/api/v1/content/zhihu_search` | 离线抓取历史回答 + 搜索兜底 | 离线 450 次/日(赛前跑完)+ 运行时 < 100 次/日 |
| `/api/v1/content/hot_list` | 首页"今日热榜话题" banner | 4 次/日(cron)+ buffer,远低于 100 限额 |
| `/v1/chat/completions`(直答) | 2029 预测生成 + 搜索兜底时的浅层时光轴生成 | < 50 次/日 |
| `POST /openapi/publish/pin` | 海报一键发布到知乎想法 | 每次用户点发布触发 1 次 |
| OAuth | 仅发布时触发登录,其他页面游客可访问 | 按需 |
| 刘看山 IP | 海报角落可选"by 刘看山推荐"印章 | 轻度使用,蹭 IP 分 |

## 七、错误处理与降级

| 场景 | 降级策略 |
|---|---|
| 直答 API 超限 | 降级到 DeepSeek / Haiku,在预测卡上加"AI 模型自动切换"小字 |
| 站内搜索 API 超限 | 搜索兜底通道提示"每日兜底名额已用完,请浏览精选话题" |
| 圈子发帖失败 | 海报自动下载到本地,提示"可手动发布" |
| LLM 输出格式错误 | JSON schema 校验 + 重试 1 次 + fallback 到预生成模板文案 |
| 用户搜索的话题无数据 | 展示"这个话题还没被时光答收录,我们已加入队列。要不要看看这些相近话题?" |
| 海报生成失败 | fallback 到纯文字卡片 |

## 十二、路演策略(5/16)

### 3 分钟 Demo 脚本

## 附录 A:项目命名由来

- **时光答** = 时光机 + 回答(双关);取名呼应"有问题,就会有答案",只是把答案放回了它诞生的年代。
- 英文备名:**ChronoQ**(Chrono + Q&A)或 **Timeline of Answers**。

## 附录 C:参考设计语言

- 视觉基调:**杂志感 + 档案馆感**
- 字体:中文思源宋体(标题)+ 思源黑体(正文)
- 色板:
  - 主色 `#0B1026` 深夜蓝(时间纵深感)
  - 次色 `#F4E4BA` 旧纸米黄(档案感)

- 布局:留白大,长滚动,每一屏一个视觉锚点
